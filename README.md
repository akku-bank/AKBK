## 3. Convention & Rules

원활한 협업과 일관된 코드 이력 관리를 위해 아래의 Git Commit 컨벤션을 준수합니다.

### Commit Type

| Type | Description |
| --- | --- |
| `feat` | 새로운 기능 추가 |
| `fix` | 버그 및 오타(typo) 수정 |
| `refactor` | 코드 리팩토링 (기능 및 로직 변화 없음) |
| `design` | CSS 등 사용자 UI/UX 변경 |
| `comment` | 주석 추가 및 변경 |
| `style` | 코드 포맷팅, 세미콜론 누락 등 (코드 로직 변경 없음) |
| `test` | 테스트 코드 작성 및 수정 |
| `chore` | 기타 변경사항 (빌드 스크립트 수정, assets, 패키지 매니저 등) |
| `init` | 프로젝트 초기 세팅 |
| `rename` | 파일/폴더명 수정 및 이동 |
| `remove` | 파일 삭제 |
| `docs` | 문서 작성 및 수정 (README 등) |
| `build` | 빌드 환경 및 설정 파일 수정 |
| `ci` | CI/CD 설정 파일 수정 |

### Commit Message Format

```text
type: 작업 내용 요약

```

*(예시)*

* `feat: 카카오 소셜 로그인 API 연동`
* `fix: 결제 페이지 렌더링 오류 수정`
